O principal problema que o Exponential Backoff com Jitter resolve em tentativas de repetição de falha.

Resolve o 'thundering herd problem' ao adicionar aleatoriedade aos intervalos entre as novas tentativas. Isso evita que todos os clientes tentem novamente ao mesmo tempo, sobrecarregando o serviço de destino.

Armazenamento efêmero disponível para uma função AWS Lambda.

O diretório `/tmp`. Ele é um armazenamento temporário e efêmero que pode ser usado para arquivos de até 10 GB durante a execução da função.

A operação mais eficiente do DynamoDB para buscar itens pela chave de partição.

A operação `Query`. Ela é otimizada para buscar itens usando a chave primária (chave de partição ou ambas, se houver uma chave de ordenação), tornando-a mais rápida e barata que um `Scan`.

A estratégia de cache do ElastiCache que garante que o cache seja atualizado simultaneamente com o banco de dados principal.

Write-Through. Cada gravação no banco de dados é imediatamente refletida no cache, garantindo alta consistência.

A principal função de um Amazon Cognito User Pool.

Autenticação. Ele atua como um diretório de usuários para gerenciar a autenticação e o login dos usuários de uma aplicação (quem o usuário é).

A principal função de um Amazon Cognito Identity Pool.

Autorização. Ele fornece credenciais temporárias da AWS para que os usuários autenticados possam acessar outros serviços da AWS (o que o usuário pode fazer).

Prática recomendada pela AWS para armazenar credenciais sensíveis, como senhas de banco de dados.

Usar o AWS Systems Manager (SSM) Parameter Store com o tipo `SecureString`. Ele armazena as informações de forma criptografada usando o KMS.

A ferramenta da AWS que simplifica a definição de infraestrutura serverless (Lambda, API Gateway, DynamoDB) usando uma sintaxe abreviada.

AWS Serverless Application Model (SAM). É uma extensão do CloudFormation focada em aplicações serverless.

A funcionalidade do CloudFormation que identifica quando os recursos de uma pilha foram alterados manualmente fora do template.

Drift Detection (Detecção de Desvio). Ela detecta 'desvios' entre o estado real dos recursos e a configuração definida no template do CloudFormation.

Serviço da AWS usado para rastreamento de ponta a ponta de requisições em uma aplicação distribuída.

AWS X-Ray. Ele fornece um mapa de serviços e rastreamentos detalhados para ajudar a identificar gargalos e latência.

O parâmetro de configuração do SQS que permite que um consumidor espere por uma mensagem, reduzindo o custo e o número de chamadas de API vazias.

`MessageWaitTimeSeconds`. Um valor maior que zero habilita o 'long polling', onde a chamada de `ReceiveMessage` espera por mensagens antes de retornar.

A principal diferença entre SQS Standard e SQS FIFO.

SQS FIFO garante a ordem de entrega das mensagens (primeiro a entrar, primeiro a sair) e o processamento 'exactly-once'. SQS Standard não garante a ordem e pode ter duplicações.

O que é uma DLQ no SQS e qual o seu propósito?

DLQ (Dead-Letter Queue) é uma fila para onde as mensagens são enviadas depois de falharem no processamento um número configurável de vezes. Seu propósito é isolar mensagens problemáticas e evitar que elas bloqueiem a fila principal.

Um caso de uso para um Global Secondary Index (GSI) no DynamoDB.

Permite realizar consultas (`Query`) em atributos que não fazem parte da chave primária da tabela principal. Ele é útil para acessar dados de maneira diferente da chave principal, usando chaves de partição e de ordenação alternativas.

O que são 'Service Quotas' na AWS?

São limites padrão definidos pela AWS para o uso de seus serviços. Por exemplo, o número máximo de instâncias EC2, ou o limite de requisições por segundo para uma API. Eles ajudam a prevenir abuso e proteger a estabilidade dos serviços.
